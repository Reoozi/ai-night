This system implements exactly what was requested:

- ✅ **Receives user questions** - Text input for natural language queries

## 🔍 How to Use

### **Step 1: Upload Documents**

1. Go to http://localhost:3000
2. Click **"📤 Upload & Index"** tab
3. Enter:
   - **Title**: e.g., "Leave Policies"
   - **Category**: e.g., "HR"
   - **Content**: Paste document text
4. Click **"📤 Index Document"**

**Note:** Documents are split into 500-token chunks for better search

### **Step 2: Search Documents**

1. Click **"🔎 Search Documents"** tab
2. Type your question:
   - "How many days of leave per year?"
   - "When is salary paid?"
   - "What are performance ratings?"
3. Select "Use reranking" (optional, improves quality)
4. Click **"🔍 Search"**

### **Step 3: View Results**

You'll see the **Top 3 Results**:

```
✨ Result #1 - Leave Policy
📊 Similarity: 87.3%
   Rerank Score: 0.92 (if enabled)
📄 Content: [Relevant passage...]
```

Each result shows:

- **Fragment text** - The actual relevant passage
- **Similarity score** - How relevant it is (0-100%)
- **Rerank score** - Additional relevance scoring (if enabled)
- **Metadata** - Title, category, source

---

## 📊 Understanding the Scores

### **Similarity Score** (0-100%)

- Generated by cosine similarity in vector space
- **80%+** = Very relevant
- **60-80%** = Somewhat relevant
- **<60%** = Less relevant

### **Rerank Score** (with Cross-Encoder)

- More sophisticated relevance measure
- Useful when cosine similarity alone isn't enough
- Can reorder results for better quality
- **Adds 200-500ms latency**

## 🔌 API Endpoints

### Index Documents

```bash
curl -X POST "http://127.0.0.1:8000/rag/index" \
  -H "Content-Type: application/json" \
  -d '{
    "documents": [{
      "content": "Your document here...",
      "title": "Document Title",
      "category": "Category"
    }],
    "use_reranker": false
  }'
```

### Search Documents

```bash
curl -X POST "http://127.0.0.1:8000/rag/search" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Your question here",
    "top_k": 3,
    "use_reranker": true
  }'
```

### Get Statistics

```bash
curl "http://127.0.0.1:8000/rag/stats"

# Response:
{
  "total_documents": 3,
  "embedding_model": "all-MiniLM-L6-v2",
  "embedding_dimension": 384,
  "reranker_model": "cross-encoder/ms-marco-MiniLM-L-12-v2"
}
```

### Clear Database

```bash
curl -X DELETE "http://127.0.0.1:8000/rag/clear"
```

## 🐛 Troubleshooting

### "Connection refused" Error

**Problem:** Backend not running

**Fix:**

```bash
# Check if backend is running
netstat -an | findstr "8000"  # Windows
# or
lsof -i :8000  # Mac/Linux

# If not running:
python app/main.py
```

### "PostgreSQL connection error"

**Problem:** Database not running

**Fix:**

```bash
# Start PostgreSQL
cd docker
docker-compose up -d

# Verify it's running
docker ps | grep postgres
```

### "No module named 'pgvector'"

**Problem:** Dependencies not installed

**Fix:**

```bash
pip install -r requirements.txt
```

### "No results found"

This means:

1. No documents in database yet - Upload some!
2. Query doesn't match documents - Try different words
3. Documents are too different - Increase `top_k` value

## 🔑 Key Technical Details

### Document Processing

1. **Input:** Raw text document
2. **Splitting:** RecursiveCharacterTextSplitter (500 tokens, 50 overlap)
3. **Embedding:** sentence-transformers (all-MiniLM-L6-v2)
4. **Storage:** PostgreSQL pgvector
5. **Indexing:** IVFFlat for fast search

### Search Process

1. **Query Input:** User question
2. **Query Embedding:** Generate vector for question
3. **Similarity Search:** Cosine similarity against all documents
4. **Ranking:** Sort by similarity score
5. **Reranking** (optional): Cross-encoder for refinement
6. **Results:** Return top 3 with scores

### Models Used

- **Embedding:** `all-MiniLM-L6-v2` (384 dimensions, fast, good quality)
- **Reranker:** `cross-encoder/ms-marco-MiniLM-L-12-v2` (better ranking)
- **Database:** PostgreSQL 15 with pgvector extension
